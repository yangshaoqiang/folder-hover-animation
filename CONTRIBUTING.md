# Contributing

## Development

- Open `index.html` directly in a browser for local preview.
- Keep the project dependency-free unless there is a strong reason to add tooling.
- Preserve the current folder interaction unless a change is intentional and documented.

## Style

- Prefer small, focused changes.
- Keep animation values readable and grouped in CSS.
- Reuse the existing asset structure under `assets/`.

## Pull Requests

- Describe what changed and why.
- Include before/after screenshots or a short screen recording for animation changes.
- Mention any parameter changes to layer movement, delay, or lid angle.
